# 로그인 버튼 너비 통일 (공식 이미지 사용)

CLAUDE.md의 코드 스타일 규칙(early return, 삼항연산자 금지, else if 금지)을 반드시 지켜서 작성해줘.

---

## 배경

카카오/구글 로그인 버튼은 각 브랜드의 공식 리소스 이미지를 그대로 사용해야 해 (버튼 심볼/레이블/컬러를 임의로 재구성하면 브랜드 가이드 위반). 단, 두 버튼의 "표시 너비"는 동일하게 맞춰야 하고, 이때 각 이미지 고유의 가로세로 비율은 유지해야 해 (비율을 억지로 왜곡하면 안 됨).

---

## Step 1. 에셋 파일 확인

`assets/icons/` 폴더를 확인해서 카카오, 구글 버튼 이미지 파일명을 찾아줘. 파일명에 "kakao"가 포함된 파일과 "google"이 포함된 파일을 각각 찾아서 사용해줘.

---

## Step 2. pubspec.yaml 에셋 등록

`pubspec.yaml`의 `flutter: assets:` 항목에 `assets/icons/`가 등록되어 있는지 확인하고, 없으면 추가해줘.

```yaml
flutter:
  assets:
    - assets/icons/
```

---

## Step 3. LoginButtons 위젯 수정

`lib/features/auth/` 안에서 로그인 버튼을 렌더링하는 위젯(기존 `LoginButtons` 관련 파일)을 찾아서 아래 요구사항대로 수정해줘.

### 요구사항

- 버튼 통일 너비: **240px**
- 각 버튼은 `AspectRatio` 위젯으로 감싸서, 원본 이미지의 가로세로 비율을 유지한 채 너비만 240px로 맞춰줘.
- 원본 이미지의 실제 픽셀 크기(가로/세로)를 확인해서 `aspectRatio` 값을 정확히 계산해줘. (예: 원본이 400x56이면 aspectRatio는 400/56)
- 두 버튼 사이 간격은 12px, `Column`으로 세로 배치.
- 각 버튼은 `GestureDetector` 또는 `InkWell`로 감싸서 탭 이벤트를 받되, 이미지 자체의 색상/모양은 변형하지 말 것 (예: `ColorFiltered`, `Opacity` 등으로 브랜드 색상 왜곡 금지).

### 예외 처리

- `Image.asset`에 `errorBuilder`를 추가해서, 파일이 없거나 로드 실패 시 240px 너비의 회색 placeholder 컨테이너를 대신 표시하고 콘솔에 `debugPrint`로 에러 로그를 남겨줘.

### 코드 스타일

- 위젯은 `StatelessWidget`으로 작성 (상태 없음)
- 매직 넘버(240, 12 등)는 위젯 상단에 `const` 변수로 선언해서 이름 붙여줘 (예: `_buttonWidth`, `_buttonGap`)
- early return 패턴 유지, 조건문에 삼항연산자/else if 사용 금지

---

## Step 4. 검증 요청

수정 완료 후 아래 사항을 확인해서 알려줘.

- [ ] 두 버튼의 실제 렌더링 너비가 동일한지 (240px)
- [ ] 각 버튼의 세로 길이가 서로 다를 수 있음 (원본 비율이 다르면 정상) — 이는 의도된 동작이니 문제 아님
- [ ] `flutter analyze` 실행 결과 에러/경고 없는지
- [ ] 이미지 파일이 실제로 존재하고 정상 로드되는지 (디버그 모드에서 화면 확인 필요함을 안내)

작업 완료 후 `/commit-push` 명령어로 커밋 및 푸시 진행해줘.
